3)
Параметр charset у заголовка Content-Type требуется для определения кодировки. 
Если charset будет не задан или будет задан не верно, то результат будет зависеть от браузера.
Браузер может сам понять, какая нужна кодировка, в противном случае пользователь увидит 'крокозябры'.
 
в примере  http://localhost/lw1/all_begin_from_http.html

nginx в теле ответа не указывает параметр charset у заголовка Content-Type,
браузер не может определить кодировку, видим нечитаемый текст

Существует несколько способов исправить ситуацию:
1)Указать HTTP заголовок Content-Type .
  Можно на сервере добавить кодировку по умолчанию utf-8
  тогда он добавит заголовок Content-Type параметр charset
2)В теле html докумнета добавить мета тэг.
  Ответ сервера по прежнему не будет содержать параметра charset,
  но браузер определит кодировку благодаря благодаря подсказке
3)Передавать в теле ответа BOM.
  Можно html документ перевести в кодировку с BOM,
  результат аналогичен п.2.

3)
Параметр charset у заголовка Content-Type требуется для определения кодировки. 
Если charset будет не задан или будет задан не верно, то результат будет зависеть от сервера.
Сервер может сам 'догадаться' какая нужна кодировка, в противном случае пользователь увидит 'крокозябры'.
(php сервер догадался, nginx нет)
 
в примере  http://localhost/lw1/all_begin_from_http.html
не хватает тега, указывающего кодировку <meta charset='UTF-8'> 
без него charset у заголовка Content-Type не определен, 
а на сервере nginx по усолчанию для файлов с расширением .html,
видимо, другая кодировка

Чтобы исправить, нужно добавить тег <meta charset='UTF-8'> 
(весьма желательно в header)
Не будет лишним на сервере поставить utf-8 кодировку по умолчанию
charset UTF-8; в блок сервера
